## ✅ 修复方案

### 1. 移除模拟数据逻辑

**修改文件**: `presenters/brief_generator_enhanced.py`

**修改前**:
```python
# 如果所有周期都没有数据，使用模拟数据（演示用）
total_predictions = sum(s['预测次数'] for s in stats.values())
if total_predictions < 3:
    logger.info("[准确率统计] 数据不足，使用模拟数据")
    stats = {
        '7 天': {'预测次数': 12, '正确次数': 7, '准确率': 58.3},
        '30 天': {'预测次数': 45, '正确次数': 26, '准确率': 57.8},
        '90 天': {'预测次数': 128, '正确次数': 71, '准确率': 55.5},
    }
```

**修改后**:
```python
# 直接返回真实数据，不使用模拟数据
logger.info(f"[准确率统计] 从 2026-04-09 开始计算真实数据，当前共 {stats['7 天']['预测次数']} 条已验证记录")
return stats
```

### 2. 修复键名不一致问题

**问题**: 键名生成时使用 `f'{days}天'`（无空格），但访问时使用 `stats['7 天']`（有空格）

**修复**:
```python
# 使用正确的键名（带空格，与模板一致）
key = f'{days} 天'
stats[key] = {
    '预测次数': total,
    '正确次数': correct,
    '准确率': round(accuracy, 1)
}
```
