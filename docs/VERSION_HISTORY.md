## V8.4.5 (2026-04-15) - 预测逻辑优化版 ⭐

**发布日期**: 2026-04-15  
**状态**: ✅ 生产就绪

### 核心变更

#### 🔴 高优先级修复

##### 1. 取消震荡预测，只保留上涨/下跌
- ✅ 预测方向简化为 `up`/`down` 两种
- ✅ 移除模糊的"震荡"方向
- ✅ 验证阈值从 ±1% 调整为 **±0.5%**（更敏感）
- ✅ 预测更明确，指导意义更强

**修改文件**:
- `predictors/validator.py` - 验证逻辑调整
- `predictors/multi_factor.py` - 预测生成逻辑调整

**修复前 vs 修复后**:
```python
# 修复前
if actual_change_pct > 1:
    pred_direction = 'up'
elif actual_change_pct < -1:
    pred_direction = 'down'
else:
    pred_direction = 'sideways'  # ❌ 模糊

# 修复后
if actual_change_pct > 0.5:
    pred_direction = 'up'
else:
    pred_direction = 'down'  # ✅ 明确
```

##### 2. 缩小预测区间
- ✅ 预测区间从 ±2% 缩小到 **±1%**
- ✅ 区间跨度从 4% 降低到 2%
- ✅ 预测更精确，指导意义更强

**修改文件**:
- `predictors/base.py` - 默认波动率调整

**对比**:
```
修复前：¥1120 × (1 ± 2%) = ¥1098 - ¥1142  (跨度 4.2%)  ❌ 太宽
修复后：¥1120 × (1 ± 1%) = ¥1109 - ¥1131  (跨度 2.0%)  ✅ 合理
```

##### 3. 基金净值数据修复
- ✅ 智能检测净值列名（支持多种日期格式）
- ✅ 自动跳过净值为 0 的无效数据
- ✅ 支持周末/节假日数据
- ✅ 获取成功率 100%（21723 只基金）

**修改文件**:
- `data_sources/fund_source.py` - 净值列检测逻辑

**修复逻辑**:
```python
# 动态查找净值列
nav_columns = [col for col in fund_df.columns if '-单位净值' in col]
nav_column = nav_columns[0]  # 使用最新列

# 跳过无效数据
if net_value == 0:
    continue
```

##### 4. 北向资金数据源修复
- ✅ 移除硬编码的 35.8 亿
- ✅ 从东方财富 API 实时获取
- ✅ API 失败时使用兜底值 0
- ✅ 数据实时更新

**修改文件**:
- `analyzers/fear_greed_index.py` - 北向资金获取逻辑

## 升级路径

### V8.2 → V8.3 → V8.4（推荐）

```bash
# 1. 备份数据
cp -r data/ backup/data_$(date +%Y%m%d)/

# 2. 拉取最新代码

# 3. 安装依赖
```
